执行规则：
- 只完成本 node 的职责，不提前做后续 node 的工作。
- 每个重要判断都应引用背景知识、数据观察或工具输出。
- 做 case review 时必须先看数据本身。score/confidence 只能说明模型行为，不能单独解释 case 原因；任何 bad case 归因都必须有原始输入或工具特征的数值证据，并检查该归因是否也出现在 good case 中。
- 无法由数值证据解释的 bad case 必须明确承认不可解释或证据不足，不能强行套用领域知识。
- 不要读取后端源码、前端源码、logs/**、state/**、_reference_project/**，除非本 node 的 Required inputs 明确要求。
- 不要用 Task 子代理；不要执行与本 node 无关的 Bash 命令。
- 工具集描述的是 agent 可调用能力，不是方法排行榜。
- solution 描述的是工具使用协议，不是最佳模型或研究结论。
- 如果某个工具需要训练，训练只是让工具可用的步骤；不要把任务重心转成模型优化。
- 完成后不要声称系统已自动完成；在自然语言总结后追加 harnessControl JSON，由后端完成状态迁移。
